# 📈 Monthly Sales Trend Analysis

## 📌 Project Overview
This project analyzes monthly sales trends using SQL aggregation functions in MySQL. The analysis focuses on monthly revenue and order volume to identify sales patterns, top-performing months, product performance, and key business insights.

## 🎯 Objectives
- Objective: Analyze monthly revenue and order volume.

## 🛠 Tools Used
- MySQL Workbench

## 📂 Dataset
The dataset contains the following columns:

- **order_id**
- **order_date**
- **amount**
- **product_id**

## 📊 Analysis Performed

### Basic Analysis
1. Calculate monthly revenue.
2. Calculate monthly order volume.
3. Analyze monthly revenue and order volume together.
4. Analyze revenue and order volume for Jan–Mar 2024.
5. Find months with highest revenue first.

### Intermediate Analysis
6. Find the month with the highest revenue.
7. Find the month with the lowest revenue.
8. Identify months where revenue exceeded ₹2000.
9. Find months with more than 3 orders.
10. Calculate average revenue per order each month.
11. Find total revenue generated in the first quarter (Jan–Mar).

### Advanced Analysis
12. Rank months based on revenue using Window Functions.
13. Identify the top 3 products by revenue.
14. Calculate total orders per product.
15. Find the top 3 months with the highest order volume.

## 🔑 SQL Concepts Used

- Aggregate Functions (`SUM()`, `COUNT()`, `AVG()`)
- Date Functions (`EXTRACT()`)
- GROUP BY
- ORDER BY
- HAVING
- WHERE Clause
- LIMIT
- Window Functions (`RANK()`)

## 📈 Key Insights

- Monthly revenue and order volume trends were analyzed.
- High-performing and low-performing months were identified.
- Quarterly revenue was calculated.
- Revenue ranking was performed using window functions.
- Top products based on revenue were identified.
- Product-wise order distribution was analyzed.
- Months with the highest order volume were determined.
